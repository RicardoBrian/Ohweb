Design Reference
0 · 핵심 원칙
색 = 상태 전용. 선택된 탭/토글 켜짐/진행률처럼 "지금 어떤 상태인지" 보여줘야 하는 곳에만 accent(또는 중립 다크)를 쓴다. 로그인·저장·제출 같은 1회성 CTA는 파랗게 채우지 않는다 — 아래 1번 참고.
1 · 버튼 위계
주요 액션(로그인/저장/제출)은 텍스트 있는 중립 다크 필. 반복 액션(수정/삭제/새로고침)은 아이콘 전용 원형 버튼, 텍스트 없음. 취소/닫기는 ghost(배경 없음). 이 셋 외의 새로운 버튼 스타일을 만들지 않는다 — 종류가 늘어날수록 통일감이 깨진다.
주요 CTA — .btn.accent (파란색 아님, 중립 다크)
보조 액션 — .btn (기본 glass), .btn.ghost, .btn.danger
반복/자명 액션 — .icon-action (텍스트 없이 아이콘만)
.icon-action {
  width: 34px; height: 34px; border-radius: 50%; border: none; background: none;
  display: inline-flex; align-items: center; justify-content: center;
  color: var(--sub);                 /* 기본은 회색 */
}
.icon-action:hover { background: var(--glass); color: var(--text); }
.icon-action.danger { color: #ef4444; }   /* 삭제류만 항상 빨강 */
2 · 리스트 행
반복되는 데이터 행(학생/게시글/시험 등)의 액션은 전부 아이콘 전용. "수정" "삭제" 텍스트 버튼을 행마다 반복하지 않는다.

김철수 · 3학년 2반
3 · 그룹 인셋 리스트 (폼 기본형)
label + input을 카드 하나에 각각 흩어놓지 말고, iOS 설정 앱처럼 "라벨 왼쪽 · 값 오른쪽"인 행을 hairline으로 이어붙인다. select에는 우측에 셰브런(⌃⌄) 힌트를 넣어 "누를 수 있는 요소"임을 알린다.

4 · 히어로 숫자
"지금 가장 중요한 값 하나"는 화면 최상단 큰 숫자로. 그 아래 보조 정보는 작은 캡션 한 줄로 압축한다. 같은 크기 카드를 여러 개 나열하는 것보다 훨씬 위계가 분명하다.
총 지출
₩1,234,000
지출 12건 · 참여자 5명
5 · 세그먼트 / 탭 — 선택 상태만 색상
탭의 "선택됨"은 이 시스템에서 색을 써도 되는 자리다. 단, 선택 안 된 탭은 완전 무채색이어야 하고, 화면에 다른 색이 또 있으면 안 된다(색은 한 화면에 최대 1곳).
6 · 파비콘 / 앱 아이콘
다크(#1d1d1f) 라운드 사각 타일(border-radius ≈ 23%) + 흰색 라인 글리프 (stroke만, fill 없음, stroke-width ≈ 3). 앱마다 글리프만 바꿔서 패밀리감을 유지한다 — 예: 관리=기어, 학생=졸업모, 정산=지갑. 
<svg viewBox="0 0 64 64">
  <rect width="64" height="64" rx="14.5" fill="#1d1d1f"/>
  <g fill="none" stroke="#ffffff" stroke-width="3.2"
     stroke-linecap="round" stroke-linejoin="round">
    <!-- 글리프 (24x24 아이콘 세트와 톤 맞추기) -->
  </g>
</svg>
<!-- head 삽입 -->
<link rel="icon" type="image/svg+xml" href="icon.svg">
<link rel="apple-touch-icon" href="apple-touch-icon.png"> <!-- 180px PNG 스냅샷 -->
7 · 문구는 최소한으로
페이지 제목 아래 설명 문장("이 페이지는 ~합니다")을 넣지 않는다. 화면 구성 자체가 설명이 되게 만든다. 버튼 라벨은 동사 하나로 압축 ("저장 & 공유 링크 받기" → "저장"). 아이콘 하나로 뜻이 통하면 텍스트를 생략하고, 애매하면(입력·제출처럼 매번 다른 값을 다루는 액션) 텍스트를 유지한다.
